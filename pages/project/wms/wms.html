# 项目介绍

# OMS

OMS为订单管理系统，为客户使用。主要分为入库管理，出库管理，费用管理，统计分析(未做)，基础设置，异常管理七大模块。

# WMS

WMS为仓库管理系统，为海外仓客服使用。主要分为入库管理，出库管理，库内管理，费用管理，客户管理，统计分析(未做)，基础设置，海外仓操作，异常管理九大模块。

# OWMS

OWMS为海外仓系统，为海外仓库工作人员使用。主要分为入库管理，出库管理，库内管理，统计分析(未做)，基础设置，异常管理六大模块。

# 账号注册

# WMS账号注册

  • OMS账号可以通过注册获得，也可以通过登录WMS系统客户管理-客户管理-新增获得OMS账号
  • WMS账号需在基础设置-组织架构管理-新增人员获得WMS账号
  • OWMS账号需在基础设置-组织架构管理-新增人员获得OWSM账号
  • 方便查看操作日志，OMS，WMS每个人需要注册自己的账号，OWMS可以使用admin账号登录
  • 开发测试环境与阿里测试环境都需注册自己的账号，方便进行测试及修复bug & 优化
  • 因操作会产生费用项，部分操作要求必须有报价单，所以方便测试，每个人需创建自己的报价单，或者创建适用于所有客户的报价单

# WMS权限说明

  1. 接口权限需在基础设置-系统管理-接口管理中新增接口
  2. 操作按钮需在基础设置-系统管理-按钮管理中新增按钮
  3. 在代码对应的按钮上增加如下指令，如v-has="'B060'"(v-has必须加单引号)

# 错误码

如果有新增的错误码，后端发过你了，需要在项目的errcode_zh.js，errcode_en.js文件中添加相应的错误码，如果后端没有发给你，测试提出了相应的bug，将该bug转给后端相应的开发人员
