    # pda打托

    # 2023-04-14

    # 需求

    第一版pda打托操作繁琐，为了解决该问题进行相关优化

    # 功能理解

    • 所以入库订单入库均必须打托，不存在未打托的情况
    • 主流程：在全部列表领取入库单 -> 单据流转到我的 -> 扫描托盘 -> 托盘是否使用 -> 否：显示托盘号相关信息，选择托盘类型 -> 扫描sku -> sku是否存在 -> sku是否与已添加的sku重复 -> 是：输入数量，累加；否：覆盖原来的sku，输入数量 -> 保存 -> 该入库单是否打托完成 -> 是：打托完成，返回主列表；否：继续打托
