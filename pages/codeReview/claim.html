    # 主讲人：朱雪萍

    # 索赔

    # 2023-06-12

    # 需求

    # 成本索赔：

    • 物流服务商成本账单费用收取错误
    • 客户预报重量尺寸与物流服务商测量尺寸不符，导致物流服务商收取错误

    # 库内索赔：

    • 库内sku数量存在差异，存在数量缺失；仓库可能发错货或者货品丢失
    • 库内商品存在破损，造成损失

    # 快递索赔：

    • 快递在发送过程中，快递丢失，或者货物破损
    • 快递运送，但是物流信息缺失（有第一枪无物流信息 ）

    # 功能理解

    # 成本索赔：

    • 账号内可设置oversize索赔规则（无忧达付款必选），导入物流服务商账单，一种方式是系统自动判断unauthorize是否收费,是的话当该订单补收完成，自动生成一条成本索赔单，并且判断是否符合oversize索赔规则并且收费了，也会生成一条成本索赔单；另一种方式是通过人工创建/移入（物流对账处移至索赔）。生成索赔单后进行定类（判断责任方，针对单个sku定类），如果是客户错误或者服务商错误，则必须上传至少四张图片（sku长，宽，高）；然后财务再进行复检下载，提交给物流服务商（线下进行），财务确认索赔后，确认是否赔付给客户。如若改索赔单没有选择客户代码，单据流转至待认领索赔订单中，客服认领后再进行定类操作，若无指定客户，则默认归入无忧达，有无忧达承担

    # 库内索赔：

    • 首先在佩服审核设置设置客服可审核最大金额（预留），客服创建库内索赔单，针对单个sku选择索赔类型（第一枪无物流，发错货，库内丢件，库内破损），判断索赔金额是否大于客服可审核金额（暂不需要，预留），客服审核，财务审核，审核时根据客户剩余方数以及欠款判断赔付方式：自动赔付，预约赔付，再按照赔付时间，赔付金额赔付，仓库未责任方时，需要赔付给客户，进行反向充值。

    # 快递索赔：

    • 客服创建快递索赔单，待服务商返回caseNumber后录入系统，财务确认索赔完成后客服确认是否赔付给客户（反向充值），财务审核是否通过
    • 索赔类型：货物损坏，货物丢件，未货到付款
